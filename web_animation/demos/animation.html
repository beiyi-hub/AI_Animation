1. 什么是 Vibe Coding?

"我只负责写 Prompt，复制粘贴代码，运行，报错，把错误扔回给 AI。我不再写具体的语法，我只管理编程的'氛围' (Vibe)。"

— Andrej Karpathy (前 Tesla AI 总监, OpenAI 创始成员)

核心概念

  • 从 Coder 到 Manager： 你不再是搬砖的工人，你是包工头。
  • 自然语言即代码： 英语（或中文）成为了新的编程语言。
  • 关注点转移： 从“这个函数怎么写”转移到“这个功能应该长什么样”。
  • 只要能跑就行： 即使代码很丑，只要通过测试且实现功能，就是好代码。
// 传统编程
function qSort(arr) { ... }

// Vibe Coding
"帮我写一个快速排序，要带可视化动画，用 React 实现。"

技术演进历程

1. Manual Coding
查文档，手写每一行代码，StackOverflow 复制粘贴。
2. Copilot Era (AI 辅助)
Tab 键补全代码，AI 是副驾驶，依然需要人懂每一行逻辑。
3. Vibe Coding (AI 主导)
Composer/Agent 模式。AI 编写整个文件/模块，人类负责审核和引导。

2. 主流开发工具

工欲善其事，必先利其器。

Cursor 行业标杆

  • 核心功能： Tab 自动补全 (Copilot++)、Composer (多文件编辑)、Codebase Indexing (全库理解)。
  • 优势： 体验最流畅，模型选择多 (Claude 3.5, GPT-4o)，社区插件丰富。
  • 适用场景： 所有类型的全栈开发。

Trae 字节跳动出品

  • 核心功能： 类似 Cursor，但不仅是 IDE，更强调 Agent (Solo 模式)。它可以自主运行终端命令、创建文件。
  • 优势： 免费 (目前)，中文支持极其友好，国内访问速度快。
  • 评价： "Cursor 的最强国产平替，且在 Agent 能力上更进一步。"

AWS Kiro 规范驱动

注：可能为您提到的 "Kiro"。

  • 核心理念： Spec-Driven Development (规范驱动开发)。
  • 功能： 不仅仅是写代码，它会先生成需求文档 (Spec)，确认无误后再生成代码。
  • 特色： "Hooks" 功能，自动在保存时更新文档或运行测试。
  • 适用： 需要严格文档和流程的企业级项目。

3. 常用命令行 (CLI) 编程

抛弃鼠标，用自然语言控制终端。

代表工具：Claude Code / Codex

$ claude "帮我检查一下当前目录的 git status，如果有修改，提交并推送到 origin main，提交信息写 'update features'"

工作原理：

  1. 用户输入自然语言指令。
  2. LLM 将其翻译为 Shell 命令 (git add ., git commit -m ...)。
  3. 工具展示即将执行的命令。
  4. 用户确认 (Y/n) -> 执行。

效率提升点

  • 自动化流程： "把 src 目录下所有 .js 文件转为 .ts，并修复类型错误。" (一行命令代替半天工作)
  • 环境配置： "帮我配置一个 Dockerfile，基于 Node 18，暴露 3000 端口。"
  • Debug： 直接将错误日志管道传输给 AI：
    npm run build 2>&1 | claude "解释这个错误并给出修复方案"

4. 什么是 MCP?

Model Context Protocol

AI 时代的 USB 接口标准

背景与起源

以前，如果你想让 ChatGPT 读取你的本地数据库，你需要写一大堆代码来“喂”给它。每个工具（Linear, GitHub, Slack, Postgres）都需要单独的适配器。

Anthropic 提出了 MCP 标准：
让所有数据源都变成统一的“插头”，所有 AI 模型都变成统一的“插座”。

架构与案例

MCP Host (Cursor/Claude)
⬇️⬆️
MCP Server (GitHub/SQLite)
应用案例：
你在 Cursor 里问："查看我上周在 Linear 上分配的任务，并在代码里找到对应的 TODO 注释。"

Cursor 通过 MCP 协议连接 Linear 获取数据，不需要把数据手动复制粘贴。

5. 什么是 Agent (智能体)?

从“聊天机器人”进化为“数字员工”

核心概念：循环 (Loop)

感知 (Perception) → 思考 (Brain) → 行动 (Action) → 观察结果 (Observation) → 感知...

Chatbot: 你问它答，它没有手脚。
Agent: 你给目标，它自己分析，自己调用工具，自己改错，直到完成。

实际案例

场景： 修复一个 Bug。
Agent 的思考过程：
  1. [思考] 用户说登录报错，我要先复现。
  2. [行动] 运行 `npm test login`。
  3. [观察] 测试失败，报错 "500 Internal Server Error"。
  4. [思考] 可能是后端接口挂了，我要检查 `auth.js`。
  5. [行动] 读取 `auth.js` 源码。
  6. [思考] 发现漏了一个 await。
  7. [行动] 修改代码，保存。
  8. [行动] 再次运行测试... 通过！

6. 什么是 Skills?

赋予 Agent “手”和“脚”

含义与演变

定义： Skills (或 Tools) 是封装好的函数，Agent 知道如何调用它们。

{
  "name": "send_email",
  "description": "Send an email to a user",
  "parameters": { "to": "string", "body": "string" }
}

演变： 从简单的计算器工具，到现在可以操作浏览器 (Puppeteer)、操作数据库、部署云服务。

应用效果

  • 提升能力： 原本 LLM 不知道当前时间，有了 `get_time` skill 就能知道了。
  • 优化流程：
    给 Agent 一个 `deploy_to_vercel` 的 Skill。
    指令："写完代码后自动帮我部署上线。"
    结果：代码写完 -> 测试通过 -> 自动调用部署接口 -> 返回上线 URL。

7. 综合案例：从零构建番茄钟

Vibe Coding 全流程演示

Step 1: 前端 UI 设计 (Vibe 模式)

Prompt:
"用 React + Tailwind CSS 帮我写一个现代极简风格的番茄钟。背景要是深色的，中间一个大大的倒计时，下面有开始、暂停、重置按钮。字体要用等宽字体。"

操作：
在 Cursor Composer 中输入 -> 接受所有代码 -> 预览。

25:00

Step 2: 核心功能与 Agent 调试

需求： "点击开始后倒计时，结束时播放一段提示音。"

遇到 Bug： 点击暂停后再开始，时间跳动异常（倒计时加速了）。

Vibe Debugging： 不要自己去查 `useEffect` 依赖。直接把 Bug 现象告诉 Trae/Cursor：
"点击暂停再开始，计时器速度变快了，好像是有多个 setInterval 在运行，帮我修复它。"

Agent 行动： 自动定位到 `useEffect` 清理函数缺失，添加 `clearInterval`，问题解决。

Step 3: 部署与维护

利用 CLI / Skills：

@Terminal 帮我打包这个项目 (npm run build)
@Terminal 检查 build 文件夹大小，如果超过 1MB 给我报警。
@Agent 使用 Vercel CLI 部署当前项目，并将生产环境 URL 存到 README 中。
🎉 项目上线完成！全程未手动编写一行 HTML/CSS。

总结

  • Vibe Coding 是心态的转变：敢于放手让 AI 写代码。
  • 工具 (Cursor/Trae) 是你的外骨骼机甲。
  • MCP & Skills 是连接世界的桥梁。
  • Agent 是你未来的数字同事。

拥抱变化，成为 AI 的指挥官！
