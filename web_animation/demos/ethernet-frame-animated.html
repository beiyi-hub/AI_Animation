Ethernet Frame Format

A visual representation of the Ethernet II frame structure with accurate byte proportions

0 bytes 15 bytes 30 bytes 45 bytes 60+ bytes
Preamble
(7 bytes)
7-byte sequence of alternating 0s and 1s (10101010) to synchronize receiver
SFD
(1 byte)
Start Frame Delimiter (10101011) indicating end of preamble
Destination MAC Address
(6 bytes)
6-byte (48-bit) hardware address of the receiving node
Source MAC Address
(6 bytes)
6-byte (48-bit) hardware address of the sending node
Type/Length
(2 bytes)
EtherType (for >1500: protocol type like IPv4=0x0800) or frame length (for ≤1500)
Data/Payload
(46-1500 bytes)
Upper layer protocol data (minimum 46 bytes, maximum 1500 bytes)
FCS
(4 bytes)
Frame Check Sequence - 32-bit CRC for error detection
Interframe Gap
(12 bytes)
Minimum idle time between frames (96 bit times, approximately 12 bytes)
