PPP Frame Format

Point-to-Point Protocol frame structure visualization with 8-bit field organization

0 bytes 5 bytes 10 bytes Variable
FLAG Fields: PPP frames use 0x7E (126 in decimal) as delimiters to mark frame boundaries, with bit stuffing used to prevent this pattern appearing in data.
Flag
(1 byte)
Start of Frame delimiter (0x7E) - marks beginning of frame
Address
(1 byte)
Usually 0xFF (broadcast address) in PPP
Control
(1 byte)
Usually 0x03 (unnumbered data frame)
Protocol
(2 bytes)
Identifies the protocol encapsulated in the information field (e.g., IPv4=0x0021)
Information
(0-1500 bytes)
Encapsulated data from the upper layer protocol
FCS
(2 bytes)
Frame Check Sequence - 16-bit CRC for error detection (optional 32-bit)
Flag
(1 byte)
End of Frame delimiter (0x7E) - marks completion of frame
